# Быстрый старт с Client Data SME

## Запуск приложения

### 1. Установка зависимостей

```bash
cd client-data-sme
npm install
```

### 2. Настройка окружения

Скопируйте файл окружения (уже создан):

```bash
cp .env.example .env
```

Основные настройки в `.env`:

- `VITE_API_BASE_URL` - URL API сервера (по умолчанию: https://srvap6229.rccf.ru:8090)
- `VITE_ENABLE_MOCK_DATA` - использование mock данных (true/false)

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

### 4. Сборка для продакшена

```bash
npm run build
```

### 5. Предварительный просмотр сборки

```bash
npm run preview
```

## Тестирование приложения

### Поиск клиентов

1. Перейдите на вкладку "Поиск клиентов"
2. Используйте примеры для тестирования:
   - **Юридическое лицо**: ИНН 7713123456 или 7713987654
   - **ИП**: ИНН 123456789012 или 987654321098

3. Выберите дополнительные данные для отображения:
   - Краткий набор: основная информация о клиенте
   - Расширенный набор: детальная информация

4. Нажмите "Поиск клиентов"
5. Кликните на результат для просмотра деталей

### Массовая выгрузка

1. Перейдите на вкладку "Массовая выгрузка"
2. Добавьте данные:
   - Загрузите файл Excel/CSV или
   - Введите данные вручную

3. Формат ручного ввода:

```
ИНН: 7713123456
ИНН: 123456789012, Счет: 40817810099910004312
ОГРН: 1027700132195, Краткое наименование: РОМАШКА
```

4. Выберите набор данных для выгрузки
5. Укажите дату актуальности
6. Нажмите "Запустить выгрузку"
7. Скачайте результат в формате Excel

## Тестовые данные

Приложение работает с тестовыми данными по умолчанию (`VITE_ENABLE_MOCK_DATA=true`).

### Доступные клиенты:

- **ООО "РОМАШКА"** (ЮЛ, ИНН: 7713123456)
- **ООО "РОМАШКА ПЛЮС"** (ЮЛ, ИНН: 7713987654)
- **Иванов Иван Иванович** (ИП, ИНН: 123456789012)
- **Петров Петр Петрович** (ИП, ИНН: 987654321098)

### Связанные лица:

Для каждого клиента доступны связанные лица из БД AML SME.

## Подключение к реальному API

1. Отредактируйте `.env` файл:

```env
VITE_API_BASE_URL=https://your-api-server:8090
VITE_ENABLE_MOCK_DATA=false
```

2. Убедитесь, что API сервер доступен и поддерживает следующие эндпоинты:

- `POST /api/v1/clients/search` - поиск клиентов
- `GET /api/v1/clients/related?inn={inn}` - связанные лица
- `POST /api/v1/export/mass` - массовая выгрузка
- `POST /api/v1/export/upload` - загрузка файла

## Решение проблем

### Проблема: "Cannot find module"

**Решение**: Проверьте импорты и убедитесь, что пути правильные. Используйте `@/` для импорта из src.

### Проблема: "ESLint errors"

**Решение**: Запустите `npm run lint:fix` для автоматического исправления.

### Проблема: "Build fails"

**Решение**: Проверьте версии Node.js (требуется 18+) и переустановите зависимости.

### Проблема: "API requests fail"

**Решение**: Убедитесь, что `VITE_API_BASE_URL` настроен правильно и API сервер доступен.
