Изменение 1. Сделать интерфейс для Client data SME.
Сделать интерфейс для Client data SME, по типу http://dca.rccf.ru:8080/client-data-service . С возможностью поиска клиента, как минимум по полям: ИНН, ОГРН, Название (часть названия), номер клиента в наших базах, номер счета, ФИО (для ИП).

Существующий сервис https://srvap6229.rccf.ru:8090/swagger-ui/index.html позволяет искать клиента по всем указанным критериям. Дополнительно есть альтернативная возможность искать клиента через сервис URSA (в результате получать информацию в виде Excel), см. описание здесь: https://dit.rencredit.ru/confluence/pages/viewpage.action?pageId=366315015.

Учесть:

наличие 2 видов клиентов: ИП, ЮЛ + по ЮЛ связанные лица (ФЛ или ИП), доп инфо
наличие нескольких источников данных: получены из БД, получены из ручного ввода
наличие нескольких статусов клиентов (действующий- активный, бывший - неактивный)
Изменение 2. Сделать возможной массовую выгрузку данных по клиентам в отчет

Необходимо сделать интерфейс для выгрузки массовой выгрузки клиентских данных.
Режим работы:
1.1 Методолог загружает файл с входными данными по клиентам и дополнительными данными, которые необходимо собрать. Необходимо заранее указать шаблон файла с зафиксированными заголовками, которые потом сможет обрабатывать интерфейс.
1.2 Методолог вводит список входных данных напрямую в интерфейсе и указывает дополнительные данные, которые необходимо собрать.
2. По входным данным производится поиск клиента в Client Data SME, и собираются дополнительные данные, указанные в запросе.
3. Формируется файл, содержащий входные данные и дополнительные данные.

Входные данные (обязательно указан хотя бы один, может быть больше):

1).ИНН
2).Счет
3).ОГРН
4).Краткое наименование
5).ФИО (для ИП)
6).Дата актуальности (если не указано, то текущая дата)
//7).Дополнительные данные для сбора (набор можно выбрать)

Краткий набор:

1).Наименование клиента
2).ИНН
3).ОГРН
4).КПП
5).Счет (счета) + статус (открыт/закрыт)
6).Адрес регистрации
7).Адрес фактический
8).ЕИО: ФИО
9).Бенифициар: ФИО
10).Дата регистрации 
11).Основной ОКВЕД

Расширенный набор:
1). Наименование клиента
2). ИНН
3). ОГРН
4). КПП
5). Счет (счета) + статус (открыт/закрыт/блокирован) + даты открытия и закрытия + даты блокировок, если были
6). Адрес регистрации
7). Адрес фактический
8). Статус резидент/не резидент
9). ЕИО: ФИО + дата рождения + данные паспорта + ИНН + статус клиент/не клиент + адрес регистрации и фактический
10). Бенифициар: ФИО + дата рождения + данные паспорта + ИНН + статус клиент/не клиент + адрес регистрации и фактический
11). Дата регистрации
12). ОКВЭД (все)
